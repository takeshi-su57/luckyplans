# How to Deploy

All environments use the same Helm chart at `infrastructure/helm/luckyplans/` with per-environment values files. See [Helm Deployment](/docs/architecture/helm-deployment) for full design decisions.

Continuous delivery is handled by **ArgoCD** (pull-based GitOps). See [ArgoCD](/docs/architecture/argocd) for the operational guide.

## Architecture Overview

```
                     Traefik Ingress (k3d built-in)
                      │              │            │
                      │ /            │ /graphql   │ /auth/*
                      ▼              ▼            ▼
                    web:3000    api-gateway:3001
                                     │         │
                                     │ Redis   │ OIDC
                                     ▼         ▼
                                service-core  Keycloak:8080
                                     │              │
                                     ▼              ▼
                                  Redis:6379   PostgreSQL:5432
```

All services run in the `luckyplans` Kubernetes namespace.

### Install k3d

```bash
# Linux/Mac
curl -s https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash

# Windows (via Chocolatey)
choco install k3d
```

### Install cert-manager

Required for prod deployment (automatic TLS via Let's Encrypt). Not needed for local development.

```bash
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.17.1/cert-manager.yaml
kubectl -n cert-manager rollout status deploy/cert-manager
kubectl -n cert-manager rollout status deploy/cert-manager-webhook
kubectl -n cert-manager rollout status deploy/cert-manager-cainjector
```

See [TLS Certificates](/docs/architecture/tls-certificates) for full documentation.

## Local Deployment (laptop)

### Single command (full deploy)

```bash
pnpm deploy:local
```

This handles everything: cluster creation, image builds, k3d import, and `helm upgrade --install`.

After completion:

- **Frontend**: http://localhost
- **GraphQL Playground**: http://localhost/graphql

### Targeted deploy (rebuild one or more services)

```bash
# Rebuild and redeploy only the web frontend:
./infrastructure/scripts/deploy-local.sh web

# Rebuild multiple services:
./infrastructure/scripts/deploy-local.sh api-gateway web

# Helm upgrade only — for config/secret changes (no image builds):
./infrastructure/scripts/deploy-local.sh --helm-only
```

Targeted deploy builds only the specified service images, imports them into k3d, and does a `kubectl rollout restart` — much faster than a full deploy.

### Teardown

```bash
pnpm deploy:teardown
```

### Status

```bash
pnpm deploy:status
```

### Manual step-by-step

```bash
# 1. Create the k3d cluster
k3d cluster create luckyplans-local \
  --port "80:80@loadbalancer" \
  --port "443:443@loadbalancer" \
  --agents 1

kubectl config use-context k3d-luckyplans-local

# 2. Build Docker images
docker build \
  --build-arg NEXT_PUBLIC_GRAPHQL_URL="/graphql" \
  -t luckyplans/web:latest -f apps/web/Dockerfile .
docker build -t luckyplans/api-gateway:latest  -f apps/api-gateway/Dockerfile .
docker build -t luckyplans/service-core:latest  -f apps/service-core/Dockerfile .

# 3. Import images into k3d
docker pull redis:7-alpine
docker pull postgres:17-alpine
k3d image import redis:7-alpine              -c luckyplans-local
k3d image import postgres:17-alpine          -c luckyplans-local
k3d image import luckyplans/web:latest       -c luckyplans-local
k3d image import luckyplans/api-gateway:latest  -c luckyplans-local
k3d image import luckyplans/service-core:latest -c luckyplans-local

# 4. Deploy with Helm
helm upgrade --install luckyplans infrastructure/helm/luckyplans \
  --namespace luckyplans \
  --create-namespace \
  --rollback-on-failure --timeout 3m
```

## CI/CD with ArgoCD (recommended)

Prod deployments are handled by **ArgoCD GitOps**. See [ArgoCD](/docs/architecture/argocd) for the full operational guide.

### How it works

```
Push/merge to main → CI → Docker Build & Push → Update Tags → ArgoCD auto-sync → smoke tests
```

### Manual tag deployment

1. Go to **Actions → Update Image Tags → Run workflow**
2. Enter the image tag (e.g. `sha-abc1234`)
3. Click **Run workflow**

ArgoCD will auto-sync the new tag.

## Prod Deployment — first-time setup

### Prerequisites

- DNS A record: `yourdomain.xyz → <your-server-ip>`
- Ports open on firewall: 22 (SSH), 80 (HTTP/ACME), 443 (HTTPS)
- A GitHub PAT with `read:packages` and repo read access
- `CD_PUSH_TOKEN`: fine-grained PAT with Contents: read+write scope (required with branch protection)

### Setup steps

```bash
# SSH into the prod server

# 1. Create the k3d cluster
k3d cluster create luckyplans-prod \
  --port "80:80@loadbalancer" \
  --port "443:443@loadbalancer" \
  --agents 1

# 2. Install cert-manager
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.17.1/cert-manager.yaml
kubectl -n cert-manager rollout status deploy/cert-manager

# 3. Install ArgoCD
git clone https://github.com/takeshi-su57/luckyplans.git
cd luckyplans
./infrastructure/scripts/install-argocd.sh --github-token <your-github-pat>

# 4. Configure Keycloak client secret
#    Log into Keycloak admin at https://yourdomain.xyz/admin
#    → Clients → luckyplans-frontend → Credentials → Regenerate Secret
#    Copy the new client secret for the next step.

# 5. Bootstrap secrets
kubectl create namespace luckyplans 2>/dev/null || true
kubectl -n luckyplans create secret generic luckyplans-secrets \
  --from-literal=JWT_SECRET="$(openssl rand -base64 48)" \
  --from-literal=SESSION_SECRET="$(openssl rand -base64 48)" \
  --from-literal=KEYCLOAK_CLIENT_SECRET="<paste-client-secret-from-step-4>" \
  --from-literal=POSTGRES_PASSWORD="$(openssl rand -base64 32)"

# 6. Wait for ArgoCD auto-sync
# Monitor progress at https://yourdomain.xyz/argocd
```

## Adding Secrets

### Auto-generated secrets

**`JWT_SECRET`** and **`SESSION_SECRET`** are automatically generated by the Helm chart on first install and reused across upgrades. No manual configuration needed for local dev.

For production, pre-create them in the bootstrap step above so you control the values.

### Required secrets for production

| Secret | How to obtain |
|--------|---------------|
| `JWT_SECRET` | `openssl rand -base64 48` |
| `SESSION_SECRET` | `openssl rand -base64 48` |
| `KEYCLOAK_CLIENT_SECRET` | Keycloak Admin → Clients → `luckyplans-frontend` → Credentials → Client secret (regenerate if needed) |
| `POSTGRES_PASSWORD` | Auto-generated by Helm on first install. Override via `--set secrets.POSTGRES_PASSWORD=<value>` |
| `DB_PASSWORD` | Your application database password (when DB is added) |

### Manual secrets

```bash
kubectl -n luckyplans create secret generic luckyplans-secrets \
  --from-literal=JWT_SECRET="$(openssl rand -base64 48)" \
  --from-literal=SESSION_SECRET="$(openssl rand -base64 48)" \
  --from-literal=KEYCLOAK_CLIENT_SECRET="your-keycloak-client-secret" \
  --from-literal=POSTGRES_PASSWORD="$(openssl rand -base64 32)" \
  --from-literal=DB_PASSWORD="your-db-password"
```

### Obtaining `KEYCLOAK_CLIENT_SECRET`

The Keycloak client secret is configured in the `luckyplans-frontend` client:

1. Open Keycloak Admin Console (`https://yourdomain.xyz/admin` or `http://localhost:8080/admin` for local dev)
2. Sign in with admin credentials
3. Select the **luckyplans** realm (top-left dropdown)
4. Go to **Clients** → **luckyplans-frontend**
5. Click the **Credentials** tab
6. Copy the **Client secret** value

For local dev, the realm export pre-configures the secret as `dev-client-secret` — no manual steps needed.

For production, either use the value from the realm export or click **Regenerate** to create a new one, then update the K8s secret:

```bash
kubectl -n luckyplans patch secret luckyplans-secrets \
  --type merge -p "{\"stringData\":{\"KEYCLOAK_CLIENT_SECRET\":\"<new-secret>\"}}"
# Restart the gateway to pick up the new value:
kubectl -n luckyplans rollout restart deployment/api-gateway
```

### Rotating `JWT_SECRET`

```bash
kubectl -n luckyplans delete secret luckyplans-secrets
# Trigger an ArgoCD sync — Helm will generate a new JWT_SECRET
```

## Scaling

```yaml
# values.prod.yaml
apiGateway:
  replicas: 3
web:
  replicas: 2
```

Commit and push — ArgoCD auto-syncs.

> **Warning:** Do not use `kubectl scale` on ArgoCD-managed clusters — ArgoCD self-heal will revert the change immediately.

## Viewing Logs

```bash
kubectl -n luckyplans logs -f deployment/api-gateway
kubectl -n luckyplans logs -f deployment/web
kubectl -n luckyplans logs <pod-name> --previous  # after crash
```

## Rollback

### ArgoCD-managed (prod)

**Git revert** (recommended):

```bash
git log --oneline -5
git revert <commit-sha>
git push origin main
```

### Local (no ArgoCD)

```bash
helm -n luckyplans rollback luckyplans
```
