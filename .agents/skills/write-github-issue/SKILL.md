# Write GitHub Issue

## Checklist
1. Search for duplicates first.
2. Use title format: `[Type]: concise description`.
3. One issue = one concern.
4. For bugs: add repro steps, expected vs actual, environment, logs/screenshots.
5. For features: add motivation, proposal, alternatives, acceptance criteria.
6. Add labels (type + priority) and link related issues/PRs.
